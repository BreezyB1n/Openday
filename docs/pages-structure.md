# Pages Structure — Openday (Openday)

## 路由总览 / Route Overview

```
/                          首页：项目列表 + 搜索 + 订阅入口
/programs/[id]             项目详情页
/subscribe                 订阅偏好设置页
/subscribe/verify          邮箱验证结果页
/unsubscribe               退订确认页
/admin                     管理后台首页（需 Basic Auth）
/admin/programs            项目管理列表
/admin/programs/new        新增项目
/admin/programs/[id]       编辑项目
/admin/subscriptions       订阅者管理
```

---

## 公开页面 / Public Pages

### `/` — 首页

**用途：** 平台门面，帮助访客发现项目并快速订阅。

**页面结构：**

```
┌─────────────────────────────────────────┐
│  Header: Logo + "订阅通知" CTA 按钮       │
├─────────────────────────────────────────┤
│  Hero: 平台价值主张（一句话说明）          │
├─────────────────────────────────────────┤
│  搜索栏: 关键词搜索                       │
│  筛选条: [国家▼] [学位▼] [专业▼] [状态▼] │
├─────────────────────────────────────────┤
│  项目卡片列表（Grid）                     │
│  每张卡片：学校、项目名、国家、截止日期、   │
│           状态标签（OPEN/CLOSED）         │
├─────────────────────────────────────────┤
│  分页组件                                │
├─────────────────────────────────────────┤
│  Footer: 关于 + GitHub 链接               │
└─────────────────────────────────────────┘
```

**数据需求：**
- `GET /api/programs`（SSR，支持 URL 参数同步筛选状态）

**URL 参数：** `?country=US&degree=master&field=CS&status=open&q=oxford&page=1`

---

### `/programs/[id]` — 项目详情页

**用途：** 展示项目完整申请信息，提供直达申请官链。

**页面结构：**

```
┌─────────────────────────────────────────┐
│  面包屑: 首页 > 项目列表 > [项目名]       │
├─────────────────────────────────────────┤
│  项目头部:                               │
│    学校名 + 国旗 emoji                   │
│    项目名（大标题）                       │
│    状态标签 + 截止倒计时（如"还有 23 天"） │
├─────────────────────────────────────────┤
│  申请要求卡片:                           │
│    📋 学位层级   🌍 国家   📚 专业       │
│    🗣️ 语言要求  📊 GPA    💰 申请费     │
├─────────────────────────────────────────┤
│  项目简介（description）                 │
├─────────────────────────────────────────┤
│  CTA 按钮区:                             │
│    [前往官方申请页 →]  [订阅此类项目通知]  │
├─────────────────────────────────────────┤
│  最后更新时间（小字）                    │
└─────────────────────────────────────────┘
```

**数据需求：**
- `GET /api/programs/[id]`（SSR）

**截止倒计时逻辑：**
- deadline 存在且未过期 → 显示"还有 N 天"
- deadline 已过期 → 显示"已截止"
- deadline 为空 → 显示"截止日期待定"

---

### `/subscribe` — 订阅偏好设置页

**用途：** 访客填写邮箱和偏好，提交订阅。

**页面结构：**

```
┌─────────────────────────────────────────┐
│  标题：订阅申请开放通知                   │
│  副标题：第一时间收到目标项目开放邮件      │
├─────────────────────────────────────────┤
│  表单:                                  │
│    邮箱地址 *                            │
│    学位层级（多选，≤3）:                  │
│      □ 本科  □ 硕士  □ 博士            │
│    专业方向（多选，≤5）:                  │
│      [标签选择器或下拉多选]              │
│    目标国家（多选，≤3）:                  │
│      [标签选择器]                        │
│    [提交订阅] 按钮                        │
│                                         │
│  注：提交后发送验证邮件，请检查邮箱。      │
└─────────────────────────────────────────┘
```

**数据需求：**
- `POST /api/subscriptions`（客户端提交）

**表单验证（客户端 + 服务端双重）：**
- 邮箱格式校验
- degrees：至少选 1 个，最多 3 个
- fields：至少选 1 个，最多 5 个
- countries：至少选 1 个，最多 3 个

**可选增强：** 展示当前偏好匹配的项目数量（"您的偏好匹配 XX 个项目"）

---

### `/subscribe/verify` — 邮箱验证结果页

**用途：** 用户点击验证链接后的落地页，展示验证结果。

**状态：**

```
?status=success → 显示"订阅成功！我们将在项目开放时第一时间通知您。"
?status=invalid → 显示"验证链接无效或已过期，请重新订阅。"
?status=expired → 显示"验证链接已过期（24小时有效），请重新订阅。"
```

**数据需求：** 无（纯静态展示，token 验证逻辑在 `/api/subscriptions/verify` 处理后重定向至此）

---

### `/unsubscribe` — 退订确认页

**用途：** 用户点击邮件底部退订链接的落地页。

**流程：**

```
用户访问 /unsubscribe?token=xxx
        │
        ├─ 页面加载时调用 DELETE /api/subscriptions/unsubscribe?token=xxx
        │
        ├─ 成功 → 显示"已成功退订，不再发送通知邮件。"
        └─ 失败 → 显示"退订链接无效或已过期。"
```

**数据需求：** `DELETE /api/subscriptions/unsubscribe`（客户端调用）

## 管理后台 / Admin Pages

> 所有 `/admin/*` 页面通过 `middleware.ts` 的 HTTP Basic Auth 保护。
>
> 浏览器会弹出原生认证对话框（或可自建登录页面增强体验）。

---

### `/admin` — 管理后台首页

**用途：** 数据看板，展示关键指标。

**页面结构：**

```
┌─────────────────────────────────────────┐
│  管理后台 | [项目管理] [订阅者管理]        │
├─────────────────────────────────────────┤
│  统计卡片:                               │
│    📋 项目总数: 200    🟢 开放中: 45     │
│    📧 订阅者: 500      ✅ 已验证: 420    │
│    📨 今日推送: 12     📅 本月推送: 340  │
└─────────────────────────────────────────┘
```

**数据需求：** `GET /api/admin/stats`

---

### `/admin/programs` — 项目管理列表

**用途：** 查看、筛选、管理所有项目。

**页面结构：**

```
┌─────────────────────────────────────────┐
│  项目管理                [+ 新增项目] 按钮 │
├─────────────────────────────────────────┤
│  筛选栏: [状态▼] [国家▼] [学位▼] [搜索]  │
├─────────────────────────────────────────┤
│  项目表格:                               │
│  学校 | 项目名 | 国家 | 学位 | 状态 | 截止日期 | 操作 │
│  ...  | ...    | ... | ...  | ●开放 | ...      | [编辑][推送][删除] │
├─────────────────────────────────────────┤
│  分页                                   │
└─────────────────────────────────────────┘
```

**数据需求：** `GET /api/admin/programs`

---

### `/admin/programs/new` — 新增项目

**用途：** 管理员录入新项目数据。

**表单字段：** 对应 `programs` 表所有字段（`name`、`school`、`country`、`degree`、`field`、`status`、`deadline`、`languageReq`、`gpaReq`、`fee`、`applyUrl`、`description`）

**数据需求：** `POST /api/admin/programs`

---

### `/admin/programs/[id]` — 编辑项目

**用途：** 管理员修改项目信息，并可手动触发通知推送。

**特殊功能：**
- 表单预填当前数据
- "触发推送通知" 按钮 → 调用 `POST /api/admin/programs/[id]/notify`
- 推送后展示结果（发送 N 封，跳过 N 封，失败 N 封）
- 变更历史日志展示（来自 `program_updates`）

**数据需求：**
- `GET /api/admin/programs/[id]`（页面加载）
- `PUT /api/admin/programs/[id]`（保存）
- `POST /api/admin/programs/[id]/notify`（推送）

---

### `/admin/subscriptions` — 订阅者管理

**用途：** 查看订阅者列表和验证状态。

**页面结构：**

```
┌─────────────────────────────────────────┐
│  订阅者管理                              │
├─────────────────────────────────────────┤
│  筛选: [✅ 已验证] [⏳ 未验证]           │
├─────────────────────────────────────────┤
│  表格: 邮箱 | 学位偏好 | 专业偏好 | 国家偏好 | 验证状态 | 订阅时间 │
├─────────────────────────────────────────┤
│  分页                                   │
└─────────────────────────────────────────┘
```

**数据需求：** `GET /api/admin/subscriptions`

## 路由与 API 对应关系 / Route ↔ API Mapping

| 页面路由 | 使用的 API |
|---------|-----------|
| `/` | `GET /api/programs` |
| `/programs/[id]` | `GET /api/programs/[id]` |
| `/subscribe` | `POST /api/subscriptions` |
| `/subscribe/verify` | （由 `/api/subscriptions/verify` 重定向而来） |
| `/unsubscribe` | `DELETE /api/subscriptions/unsubscribe` |
| `/admin` | `GET /api/admin/stats` |
| `/admin/programs` | `GET /api/admin/programs` |
| `/admin/programs/new` | `POST /api/admin/programs` |
| `/admin/programs/[id]` | `GET/PUT /api/admin/programs/[id]`，`POST /api/admin/programs/[id]/notify` |
| `/admin/subscriptions` | `GET /api/admin/subscriptions` |
